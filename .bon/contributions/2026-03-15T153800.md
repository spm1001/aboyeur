The mesh peer list is trustworthy — if an agent appears, it has a live WebSocket connection somewhere. What looks like "stale entries" is almost always zombie processes from inadequate cleanup. The npx→tsx→node chain means bare `kill $PID` only kills the top process; you need process group kills (`os.killpg()` or `pkill -f`) to clean up properly. mesh-claude.py already does this correctly with `preexec_fn=os.setsid` + `os.killpg(SIGTERM)`, but any ad-hoc bridge spawning (tests, manual experiments) must follow the same pattern or you'll litter the mesh with ghosts that cause "Agent not found" errors for everyone else. The mesh server is accurate; your process management is the variable.
