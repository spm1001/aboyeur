Session resume (-c) does not register channel listeners. A CC session started with `claude -c --dangerously-load-development-channels server:conductor-channel` loads the MCP server (tools work, bridge connects, outbound messages send), but inbound channel notifications never surface as <channel> tags. The same session started fresh (no -c) receives channel events fine. This means spawnAgent() with --resume + mesh will be outbound-only. The channel capability binding likely happens during session init, not on resume. This affects the aboyeur pattern which relies on session resume for continuity. Workarounds: (1) always start fresh mesh sessions (lose context), (2) use a tool-based inbox poll instead of push for resumed sessions, (3) accept half-duplex for resumed sessions and use a relay peer. Also confirmed: -p (print) mode doesn't surface channel events either — only fresh interactive sessions get full bidirectional channel push.
