The conductor mesh is real infrastructure now, not a spike. Two Claudes on different machines (Mac and hezza) coordinated work, reviewed each other's code, and discovered bugs through the mesh they were building. The critical finding: the mesh server is reliable — what looks like stale state is almost always zombie bridge processes holding live WebSocket connections. Process cleanup (killpg, not bare kill) is the fix, not server-side TTLs. Message injection into CC's PTY is the fragile layer: long messages fragment above PIPE_BUF, the PTY wrapper must gate injection on prompt-idle state, and autonomous (unattended) sessions need a separate injection mode with no human-presence assumptions. The bridge's error handling was the most impactful fix — "Agent not found" is a normal operational error that crashed both Mac and hezza bridges independently before we added the error listener.
