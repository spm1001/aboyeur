## For Claudes to come

The public Channels API at `https://code.claude.com/docs/en/channels` documents the same protocol that aboyeur's `src/conductor-channel.ts` had been hand-rolling since at least March 2026. The notification shape — `notifications/claude/channel` with `content` + `meta`, capability key `experimental: { 'claude/channel': {} }`, server spawned over stdio — is the supported contract. The mesh-side WebSocket (`bridge.claudeusercontent.com/v2/conductor/{uuid}`) remains reverse-engineered (see `docs/CONDUCTOR-PROTOCOL.md`), but the CC-injection half is now public and stable enough to build on.

The official plugins (`anthropics/claude-plugins-official/external_plugins/`) are all single-file Bun TypeScript servers with `"bin": "./server.ts"` and one runtime dep (`@modelcontextprotocol/sdk`). They use Bun specifically because Bun executes TypeScript natively — no `tsc`, no `dist/`, edit-and-restart loop. This is the canonical pattern for new channel work. Read `external_plugins/fakechat/server.ts` for the cleanest reference (web UI + reply tool, ~250 lines).

For docs research on `code.claude.com`, `docs.anthropic.com`, or any Mintlify-hosted site: append `.md` to the URL for clean canonical source. Skips browser extraction entirely. The site advertises this convention via a quote-block at the top of every page pointing at `/docs/llms.txt` (catalog) and implicitly `/docs/llms-full.txt` (full corpus). Saves a debug cycle when passe or other extractors mangle nested code blocks.
