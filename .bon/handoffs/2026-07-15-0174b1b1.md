**The test topology is split by a two-sided runtime constraint, and this is permanent.** bun has the global `WebSocket` that node 20.19 lacks → the live mesh/bridge tests MUST run under bun. node's `node:test` supports `describe()` inside `test()` which bun's compat does NOT (bun #5090) → the daemon tests MUST run under node. **Neither runner can run the whole suite**, hence `npm test` = `test:daemon` (node) + `test:channel` (bun).
