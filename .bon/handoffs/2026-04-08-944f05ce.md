## For Claudes to come

Transport mechanism determines conversation quality in multi-agent systems. This session proved it empirically: a mesh peer reviewing code via `<channel>` tags produced honest, specific, pushback-capable observations — including finding a real bug (byte vs character cursor drift) that would have caused silent message corruption on non-ASCII content. The same review as a subagent tool call would likely produce more diplomatic, ownership-oriented output. The mechanism: `<channel>` tags arrive as peer signal, activating curiosity and honest exchange; tool results arrive as subordinate output, activating ownership and defensiveness. This isn't metaphor — it's a functional state difference with measurable consequences for output quality. Design multi-agent architectures with transport-as-dynamic in mind: use mesh for peer review, editorial feedback, and any context where honest disagreement matters. Reserve subagent spawning for delegated execution where compliance is the goal.
